# Методы управление файлами и каталогами в распределенных ОС. Их преимущества и недостатки.

Имеем проблемы:
1. Поиск файла
2. Доступ к файлу из чужого хранилища

## Методы управления файлами:

### Модель загрузки и разгрузки. 

Файл передается между клиентом (памятью или дисками) и сервером целиком

Недостаток: невыгодные операции: перенести файлик 1 Гб ради изменения 1 бита
### Модель удаленного доступа. 

Происходит системный вызов на данные из файла в файловую систему. Отдаем узлу страницы, которые нам нужны.

Недостатки: узел может оказаться узким местом, если его файлы оказались востребованы

## Методы управления каталогами

### Сделать составные имена из узла+пути: hostname//path to file

Преимущества: простая реализация

Недостатки: нужно хранить список всех IP адресов; теряем прозрачность расположения; нельзя иметь файлы из разных углов в одном каталоге; при перемещении файла нужно сообщить всем об изменении адреса

### Сохранять единое дерево каталогов.

Преимущество: легче отслеживать копирование и перемещение

Недостаток: дерево может закольцеваться

### Единое пространство имен. 

Единый каталог, который одинаково выглядит на всех узлах и любые действия синхронизируется в него

Преимущество - быстрый поиск. Недостаток - БД обойдется очень дорого
