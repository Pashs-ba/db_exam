# Непрерывный файл
Выделяем каждому файлу непрерывное место на диске. Отдельно храним структуру, которая называется директорией, в ней будет имя файла, размер и место откуда начинать.
## Плюсы
- Просто
- Быстро, для непрерывного чтения
## Минусы
- Фрагментация
# Связный список
Теперь записываем файл по блокам, но в конце добавляем ссылку на следующий блок
## Плюсы
- Без фрагментации
## Минусы
- Если потеряем один блок - потеряем весь файл
- Долгий Random Access
- Неочевидный размер блока - он будет либо 1020 + 4 на ссылку и тогда придется делить размер файла на 1020, что гораздо менее удобно, либо размер будет 1024+4 и будет проблемы с адресацией
# FAT (File Association Table)
Теперь мы храним таблицу с переходами отдельно
Стандарт для флешек, из за простоты и эффективности
![[Pasted image 20240620125155.png]]
## Плюсы
- Без фрагментации
- Меньше риск потерь
- Быстрый Random Access
## Минусы
- Безопастность - если таблицу повредить то абас (решается созданием копии таблицы)
- Хранение в оперативке
