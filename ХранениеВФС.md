# Методы организации хранения данных в файловых системах: индексные дескрипторы.

## EXT2
 Теперь файловая система состоит из:
 1. superblock (хранит инфу о количестве свободных, занятых i-node чтобы сверить количество)
 2. i-node bitmap (свободные ноды, битовый вектор)
 3. block bitmap (свободные блоки, битовый вектор)
 4. i-nodes
	 1. Состоит из 128 байт
	 2. Есть метаинфа с UID, временем создания и тд
	 3. 1-12 - указатели на блоки страниц
	 4. 13, 14, 15 - косвенная, двойная и тройная косвенная адресация
 5. data

Плюсы: При помощи 13-15 можно хранить огромные файлы (4 тб), пот потере i-node не теряется, достаточно быстро, если параллелить независимые структуры.
Минусы: Для большого файла размер не влезает в ячейку с размером
## EXT3
Добавили журналирование чтобы сделать более устойчивым. Есть несколько видов журнала: какой-то, который копирует всю структуру целиком, ordered (записывает только метаинфу и делает это синхронизировано со временем) и writeback (не синхронизировано)
