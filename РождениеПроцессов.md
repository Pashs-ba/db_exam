# Методы создания процессов в различных операционных системах. Структуры данных о процессах.

## Принципы
1. Любой процесс порождается процессом. 
2. Process controll block (дескриптор процесса, структура с полями) 
	1.  Идентификация 
	2. Состояние (контекст)
	3. История

## В Линкусе 
При создании процесс либо: 
1. Дочерний класс с помощью fork() - создаст полную копию с адресным пространством, кодом и тд. Ни один дочерний процесс не будет иметь больше прав, чем его родитель, что выгодно с точки зрения безопасности. Делает exec*() - заменяет копию родительского процесса на нужный.
2. С помощью clone можно породить не дочерний, а сестринский процесс

SystemD забирает себе все процессы без родителя. Возможны зомби - родитель застрял в бесконечном цикле и не проверяет код ребёнка.
## В Виндовс 
При рождение - это запрос в process manager, в ответ PID ребёнка, невозможны зомби
