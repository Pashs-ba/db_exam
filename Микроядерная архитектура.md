Главная идея - вынести часть внешних слоев из режим ядра в режим юзера
Внутри оставляем те слои которые исполняют решения, снаружи - планировщики 
Для system call пользовательские приложения обращаются к ядру, оно находит нужный сервер и дергает его.
# Преимущества
- ## Уменьшение потребления оперативки
		Нам больше не нужно хранить жирные планировщики
- ## Надежность
		В случае отказа в серевере в user space, его можно перезапустить без существенного вреда для ОС
- ## Распределенность
		Идеально подходит для распределенной системы, ибо можно вынести серверы
# Недостатки
- ## Время работы
		Частая смена из kernal в user space
- ## Безопастность
		Любой процесс может влиять на работу серверов, что приводит к катострофическим проблемам в безопастности
